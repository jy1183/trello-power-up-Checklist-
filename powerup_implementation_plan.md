# 트렐로 파워업(Power-Up) 기반 통합 체크리스트 구현 계획

기존 대시보드(옵션 A)에서 완성한 '통합 체크리스트' 기능을 트렐로 내부의 파워업(Power-Up)으로 분리 구축하는 별도 프로젝트(옵션 B)의 구현 계획입니다.

## 1. 아키텍처 및 재사용 전략

트렐로 파워업은 트렐로 화면 안에 삽입되는 **Iframe(웹페이지)** 형태의 확장 프로그램입니다. 따라서 기존 대시보드의 백엔드 API와 프론트엔드 UI를 재사용하여 개발 시간을 크게 단축할 수 있습니다.

### ♻️ 기존 대시보드에서 재사용한 자산 목록 (현재 `reference_data/` 폴더에 복사 완료)
1. **API 백엔드 라우트 (100% 재사용)**:
   - `api/checklists/route.ts`: 전체 보드의 체크리스트를 순회하여 가져오는 로직 및 체크 항목 완료 상태(PUT) 변경 로직
   - `api/checklists/item/route.ts`: 체크리스트 항목 추가(POST) 및 삭제(DELETE) 로직
   - `api/card/route.ts`: 카드 상세 정보 호출
2. **프론트엔드 UI 컴포넌트**:
   - `page.tsx`: 체크리스트 모달창, 드래그앤드롭 기능, 날짜별 필터링 기능 등 UI 로직

---

## 2. 신규 파워업 프로젝트 개발 단계

### 1단계: 파워업 전용 Next.js 프로젝트 초기화
- 해당 폴더(`5. 트렐로 파워업(체크리스트)`) 내에 파워업 구동을 위한 새로운 Next.js 환경을 세팅합니다.
- 복사해 둔 API 라우트들을 신규 프로젝트의 `app/api/trello/...` 위치로 옮깁니다.

### 2단계: 파워업 커넥터 및 매니페스트 구성
- `public/manifest.json` 생성: 트렐로 개발자 포털에 앱을 등록하기 위한 필수 설정 파일 (아이콘, 권한 등).
- `app/connector/page.tsx` 생성: 파워업 스크립트(`p.trellocdn.com/power-up.min.js`)를 로드하여, 트렐로 상단에 **[전체 체크리스트 보기]** 버튼을 띄우게 하는 진입점 파일.

### 3단계: 모달 UI(프론트엔드) 구축
- `app/checklist-modal/page.tsx` 생성: 사용자가 트렐로에서 버튼을 클릭했을 때 화면 중앙에 팝업(Iframe)으로 뜰 통합 체크리스트 페이지.
- 기존 대시보드의 체크리스트 UI를 이곳에 이식하고, 트렐로 팝업 사이즈 제약에 맞춰 레이아웃을 다듬습니다.

### 4단계: 트렐로 개발자 콘솔 연동 (수동 작업)
- 프로젝트 완성 및 Vercel 배포 후, 사용자의 트렐로 계정으로 **Trello Developer Portal**에 접속하여 새 Custom Power-Up을 생성합니다.
- Iframe URL 커넥터에 배포된 주소(`.../connector`)를 입력하고 워크스페이스에 설치합니다.

---

## 3. 요약 및 참고 사항

이 프로젝트는 현재 폴더에서 독립적으로 진행되며, 향후 **Vercel에 새롭게 배포(Deploy)** 해야 합니다. 기존 대시보드 프로젝트와 API 통신 구조는 완벽히 동일하므로 기능적인 버그는 거의 없을 것입니다.

준비가 되시면 1단계(Next.js 프로젝트 초기화 및 세팅) 작업을 지시해 주시면 됩니다!

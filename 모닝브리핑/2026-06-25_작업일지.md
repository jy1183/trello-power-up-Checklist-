# 2026년 06월 25일 개발 일지: 트렐로-팀즈 모닝 브리핑 자동화

## 🎯 목표
트렐로 파워업의 통합 체크리스트 데이터와 액티비티(댓글)를 활용하여, 매일(월~금) 오전 9시에 팀즈(Teams) 공유 채팅방으로 모닝 브리핑을 자동 발송하는 시스템 구축.

## 🛠️ 주요 구현 내용

### 1. 단일 코드베이스 아키텍처 도입 (Next.js 백엔드 확장)
- 외부 자동화 툴(N8n, Power Automate 등)의 제약과 복잡도를 피하기 위해, 기존 파워업 Next.js 서버에 전용 API 라우트(`src/app/api/morning-briefing/route.ts`)를 신설했습니다.
- 이를 통해 기존에 개발해 둔 '체크리스트 취합' 로직을 그대로 재사용하여 속도와 데이터 일관성을 확보했습니다.

### 2. 데이터 수집 및 분류 로직 구현
- **어제 진행 업무**:
  - 트렐로 API의 `actions` 엔드포인트를 호출하여 어제 작성된 액티비티(댓글)와 완료 처리된 항목 추출.
  - 어제 마감이었으나 미완료된 과업 추적.
- **오늘 진행 업무**:
  - 오늘 마감인 체크리스트를 담당자별로 매핑하여 리스트업.
  - 워크스페이스 멤버 목록을 대조하여, 할당된 체크리스트가 없는 인원에게 `"💡 오늘의 진행할 업무를 계획해 주세요."` 메시지 노출 로직 추가.
- **내일 진행 업무**:
  - 내일 마감 예정인 체크리스트 취합.

### 3. AI 댓글 요약 모듈 연동
- `@google/genai` 패키지를 설치하고 Google Gemini API (gemini-2.5-flash)를 연동했습니다.
- 수집된 '어제 작성된 댓글 내역'을 프롬프트로 전달하여 핵심 내용만 3~4줄로 간결하게 요약하는 기능을 완성했습니다.

### 4. 팀즈(Teams) 웹훅 포맷팅
- Teams Workflows의 '채널에 웹훅 요청 게시' URL을 타겟으로 설정.
- 기본적으로 폭이 좁은 Adaptive Card의 한계를 극복하기 위해 `"msteams": { "width": "Full" }` 속성을 부여하여 가로 폭을 넓고 시원하게 확장했습니다.
- 마크다운(Markdown) 포맷으로 섹션별 가독성을 높였습니다.

### 5. 스케줄러 자동화 세팅 및 배포
- 매주 월~금 오전 9시(KST)에 서버가 스스로 API를 실행하도록 `vercel.json` 파일에 Cron Job 설정(`"schedule": "0 0 * * 1-5"`)을 추가했습니다.
- 모든 변경 사항을 GitHub `main` 브랜치에 커밋하고 푸시(Push)를 완료했습니다.

# 트렐로-팀즈 모닝 브리핑 자동화 실행 계획

트렐로(Trello)의 체크리스트와 액티비티 데이터를 수집하고 요약하여, 매일 오전 9시에 팀즈(Microsoft Teams)로 모닝 브리핑을 발송하는 자동화 시스템 구축 계획입니다.

## User Review Required

> [!IMPORTANT]
> **최종 아키텍처 확정: Next.js 백엔드 확장 + Vercel Cron (관리가 가장 용이한 대안)**
> 유지보수성과 관리 편의성을 최우선으로 고려하여, 기존 **[옵션 A(파이썬 스크립트 분리)]**와 **[절충안(백엔드 확장)]**을 비교 검토한 결과, **절충안을 기반으로 한 단일 코드베이스 전략**을 최종안으로 확정하여 제안합니다.
> 
> **결정 사유 (단일 코드베이스의 압도적 장점):**
> 1. **파편화 방지**: 별도의 파이썬 프로젝트를 만들면 관리해야 할 언어와 환경(Node.js vs Python)이 두 개로 늘어납니다. 기존 파워업(Next.js) 내부에 API를 추가하면 TypeScript 하나로 모든 코드를 통일해서 관리할 수 있습니다.
> 2. **완벽한 로직 재사용**: 파워업 서버 내부에서 동작하므로, 체크리스트를 취합하는 기존 함수를 내부적으로 그대로 재사용할 수 있어 가장 빠르고 정확합니다.
> 3. **초간단 스케줄링(Vercel Cron)**: 파워업을 Vercel에 배포한다면, GitHub Actions나 Power Automate조차 필요 없이 Vercel에서 기본 제공하는 무료 Cron Job 기능을 이용해 "매일 오전 9시에 이 API를 실행하라"고 1줄만 설정하면 끝납니다.

## Proposed Architecture & Tools

### 1. 구현 도구 (Next.js Backend API + Vercel Cron)
- **개발 언어**: TypeScript (기존 파워업 프로젝트에 통합)
- **핵심 로직 (엔드포인트 신설)**: `/api/morning-briefing` 라우트를 신설하여 모든 작업을 일괄 처리.
  - 내부 체크리스트 서비스 함수 직접 호출 (파워업 화면과 100% 동일한 데이터).
  - 트렐로 API로 어제 작성된 액티비티(댓글) 수집.
  - Google Gemini API를 호출하여 댓글 내용 요약 (추후 모듈 교체 용이).
  - 팀즈 Adaptive Card 또는 마크다운 포맷으로 브리핑 텍스트 조립.
  - 조립된 메시지를 Teams Workflows Webhook으로 전송.
- **스케줄러**: **Vercel Cron** (매일 평일 KST 09:00에 해당 API 자동 호출). (필요시 GitHub Actions로 대체 가능)

### 2. 팀즈 연동 방안 (Teams Workflows Webhook)
- 팀즈 채널에서 "워크플로(Workflows)" 앱 추가 -> "채널에 웹훅 요청 게시" 템플릿 사용.
- 발급된 Webhook URL을 환경변수에 저장하고, Next.js API에서 최종 조립된 메시지를 POST 전송.

### 3. 데이터 처리 프로세스 플로우 (호출 시점)
1. **데이터 수집**
   - 파워업 내부 로직을 통해 '오늘/내일' 체크리스트 데이터 확보.
   - 트렐로 API를 통해 대상 보드의 '어제' 활동(댓글 등) 및 완료/미완료 데이터 확보.
2. **데이터 필터링 및 그룹화**
   - **전일 과업 (어제)**: 어제 완료된 항목, 어제 작성된 댓글, 어제 마감이었으나 미완료된 항목.
   - **금일 과업 (오늘)**: 담당자별 체크리스트 할당 및 누락자("오늘 진행할 업무를 계획해 주세요") 분류.
   - **명일 과업 (내일)**: 내일 마감 체크리스트.
3. **AI 요약**
   - 수집된 '어제 작성된 댓글' 텍스트들을 Gemini API로 요약.
4. **결과 전송**
   - 팀즈에 바로 전송 가능한 포맷으로 조립 후 Teams Webhook으로 POST 발송.
